Git

Version Control

คือ Software ที่ช่วยให้การจัดการบริหาร Source Code เป็นไปอย่างง่าย โดย Key หลักของ Version Control คือ

  1. มี Version กำกับทุกครั้งที่เกิดการแก้ไขไฟล์
  2. แยกงานหลักออกเป็นส่วน ๆ (Branch) และนำมารวมกันในภายหลัง (Merge)
  3. ติดตามความคืบหน้างานได้ง่ายจากบันทึกงาน (Commit Log)

Distributed Version Control Systems

คือระบบ Version Control แบบที่มีไฟล์ต้นฉบับอยู่บน Server เรียกว่า Repository แล้วมีเครื่อง Client ทำการ Clone ไฟล์ต้นฉบับบน Server มาที่เครื่องตัวเอง โดยจะได้มาทั้งไฟล์และเวอชั่นต่าง ๆ ที่เกิดขึ้นจากการแก้ไข

Git

คือ Version Control รูปแบบ Distributed Version Control Systems โดยมีขั้นตอนการทำงานง่าย ๆ คือ

  1. แก้ไขไฟล์ใน directory
  2. stage ไฟล์ที่แก้ไข
  3. commit ไฟล์ที่ stage ไว้

Installing Git

สามารถ Download ได้จาก https://git-scm.com/download
เมื่อติดตั้งเสร็จแล้วให้ทำการเปิด Command Line ขึ้นมาเพื่อทำการ set username และ email เพื่อให้ git ใช้ระบุตัวตนเมื่อมีการ commit เกิดขึ้น โดยให้พิมพ์คำสั่งดังนี้

  1. git config --global user.name "ชื่อเรา"
  2. git config --global user.email อีเมล์ของเรา

สังเกตว่าการใส่ user.name มี double-quote ครอบ แต่ user.email ไม่ต้องมี
เมื่อเสร็จแล้วให้ทำการพิมพ์

  1. git config --list

จะพบว่ามีแถว

  1. user.name=ชื่อที่เราใส่
  2. user.email=emailที่เราใส่

ถือว่าติดตั้งได้เรียบร้อย

Basic Git

เราสามารถสร้าง Git Project ได้สองวิธีคือ

  1. สร้างจาก directory ที่มีอยู่แล้วในเครื่องด้วยคำสั่ง

    • git init เพื่อทำการสร้างไฟล์สำหรับ Repository
    • git add . เพื่อเพิ่มทุกไฟล์เข้าสู่ระบบ Repository
    • git commit -m "first commit" เพื่อทำการ commit ไฟล์ที่ได้ add เข้าสู่ระบบ

  2. Clone จาก repository ที่มีอยู่แล้ว คือเรามี Git Repository อยู่บน Server แล้วต้องการ Clone ลงมาที่เครื่องของเรา สามารถทำได้ด้วยคำสั่ง

    • git clone [url]

เมื่อเราทำการแก้ไขไฟล์ใน Repository เราสามารถเช็คว่ามีไฟล์ใดถูกเปลี่ยนแปลงบ้างได้ด้วยคำสั่ง

  1. git status

เมื่อเรามีการเพิ่ม แก้ไข หรือลบไฟล์เราสามารถ stage ไฟล์ ๆ นั้นได้ด้วยคำสั่ง

  1. git add ชื่อไฟล์ - สำหรับเจาะจง stage ไฟล์
  2. git add . - สำหรับ stage ทุกไฟล์ที่มีการเปลี่ยนแปลง

เมื่อเราทำการ stage ไฟล์ที่ถูกแก้ไขเรียบร้อยแล้ว เราสามารถทำการ commit ไฟล์ที่ถูก stage ไว้เพื่อเป็นการ update Repository ได้ด้วยคำสั่ง

  1. git commit -m "ข้อความ"

โดยไฟล์ที่ถูกแก้ไข แต่ไม่ได้ถูก stage ไว้ด้วยคำสั่ง git add จะไม่ถูกนำมา commit หมายความว่าไฟล์จะเปลี่ยนแปลงแค่ในเครื่องเราเท่านั้น

ข้อความที่ใส่ในการ commit ควรบ่งบอกถึงจุดประสงค์ที่เกิดการแก้ไข ชัดเจน กระชับ เพื่อให้ผู้อื่นที่ร่วมงานสามารถเข้าใจได้ง่าย

เมื่อทำการแก้ไขไฟล์ในเครื่องเรา แล้วทำการ git add และ git commit เรียบร้อยก็ถึงเวลาที่เราจะทำการอัพไฟล์ไปยังต้นฉบับที่อยู่บน Server หรือเรียกอีกอย่างว่า Remote Repository ซึ่งเราสามารถตรวจสอบว่ามี Remote Repository ใดบ้างที่ผูกอยู่กับ Repository ด้วยคำสั่ง

  1. git remove -v

เพิ่ม Remote Repository ได้ด้วยคำสั่ง

  1. git remote add name url
    • name คือชื่อเรียกของ remote
    • url คือ url ของ remote เช่น https://github.com/git/git.git

ลบ Remote Repository ได้ด้วยคำสั่ง

  1. git remote rm name
    • name คือชื่อเรียกของ remote ที่ต้องการลบ

หากต้องการ update ไฟล์บนเครื่องเราไปยัง Remote Repository สามารถทำได้ด้วยคำสั่ง

  1. git push ชื่อremote master

หารต้องการ update ไฟล์บนเครื่องเราให้ตรงกับ version ล่าสุดบน Remote Repository สามารถทำได้ด้วยคำสั่ง

  1. git pull ชื่อremote master
